# Auto Clicker

기존 서버시간 측정 프로그램을 켜 둔 상태에서 `/api/state` 값을 읽고, 목표 서버시각에 마우스를 1회 클릭한다.

기존 `src` 코드는 수정하지 않는다.

## 기본 실행

1. 먼저 프로젝트 루트의 `run.bat`를 실행하고 대상 URL 측정을 완료한다.
2. 클릭할 위치에 마우스를 올려 둔다.
3. `run-clicker.bat`를 더블클릭하고 목표 시간을 입력한다.

실행하면 아래 양식이 콘솔에 표시된다.

```text
20:00:00.000
20:00:00
2026-05-02 20:00:00.000
```

`TargetTime`은 기본적으로 한국 시간(KST) 기준이다. 날짜를 생략하면 오늘 날짜를 쓰고, 이미 지난 시간이면 다음 날로 잡는다.

클릭은 항상 실행 시점의 현재 마우스 위치에서 왼쪽 클릭 1회만 수행한다.

## 선행 클릭 보정

클릭 API 호출, 브라우저 처리, 사이트 반응 지연을 감안해 목표보다 조금 먼저 클릭하려면 `LeadMs`를 쓴다.

```powershell
.\run-clicker.bat -TargetTime "20:00:00.000" -LeadMs 15
```

위 예시는 서버시각 `19:59:59.985`에 클릭 API를 호출한다.

## 주요 옵션

- `-TargetTime "HH:mm:ss.fff"`: 오늘 또는 다음 KST 시각
- `-TargetTime "yyyy-MM-dd HH:mm:ss.fff"`: 지정 KST 날짜/시각
- `-LeadMs 0`: 목표보다 먼저 실행할 ms
- `-ApiBase "http://127.0.0.1:8765"`: 상태 API 주소
- `-ResyncBeforeMs 3000`: 목표 직전 재동기화 시점
- `-SpinMs 25`: 마지막 busy-wait 구간

## 주의

`Stopwatch` 기준 대기는 ms 단위에 적합하지만, Windows 스케줄링과 브라우저/사이트 처리 지연은 완전히 제어할 수 없다. 실제 환경에서 `LeadMs`를 몇 번 테스트해 보정하는 것을 권장한다.
