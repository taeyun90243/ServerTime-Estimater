# 프로젝트 전체 설명

최종 업데이트: 2026-05-03 +09:00 (KST)

이 프로젝트는 사용자가 입력한 웹사이트 URL의 서버 시각을 추정해 브라우저에 실시간 시계처럼 보여주는 Windows용 로컬 도구다. 네이비즘과 비슷한 목적이지만, 현재 구현은 외부 배포형 서비스가 아니라 PC에서 `ServerTimeProbe.exe` 또는 `run.bat`로 실행하는 로컬 HTTP 서버 + 정적 웹 UI 구조다.

## 실행 흐름

1. 사용자가 `ServerTimeProbe.exe` 또는 `run.bat`를 실행한다.
2. `src/probe.ps1`이 로컬 HTTP 서버를 `127.0.0.1:8765`에 띄운다.
3. 브라우저가 자동으로 열린다.
4. 사용자가 상단 입력창에 URL을 입력하고 `측정`을 누른다.
5. 서버가 대상 URL을 적응형(약 6초)으로 샘플링해 오프셋을 계산한다.
6. 웹 UI가 `/api/state`로 받은 기준 시각을 `performance.now()`로 보간해 표시한다.

네이버 기준 측정이 필요하면 브라우저 입력창에 `https://naver.com/`을 입력한다. 코드 내부에서는 `naver.com` 호스트를 네이버 시계 API 경로로 특수 처리한다.

## API

`GET /`

정적 UI를 반환한다. 첫 페이지 로드는 재측정을 걸지 않고, 같은 대상이 설정된 상태에서 이후 새로고침/재진입하면 재측정을 요청한다.

`GET /api/state`

현재 측정 상태를 JSON으로 반환한다. 주요 필드는 다음과 같다.

| 필드 | 의미 |
|---|---|
| `host` | 현재 대상 호스트 |
| `targetUrl` | 현재 대상 전체 URL |
| `offsetMs` | PC 기준 대비 대상 서버 시각 offset |
| `lastMeasureAt` | 마지막 측정 완료 시각 |
| `rttMedianMs` | 샘플 RTT 중앙값 |
| `sigmaMs` | 채택 후보 offset 표준편차 |
| `ci95Ms` | 95% 신뢰구간 추정치 |
| `status` | `idle`, `queued`, `measuring`, `ok`, `failed`, `stale` |
| `pcSendTimeAtMs` | API 응답 직전 PC 기준 epoch ms |
| `ntpInfo` | 참고용 PC-NTP skew 정보 |

`POST /api/target`

브라우저에서 새 URL을 설정한다. JSON 본문은 `{ "url": "https://example.com/" }` 형식이다. 스킴이 없으면 `https://`를 자동으로 붙인다. `http`와 `https`만 허용한다.

요청이 성공하면 상태가 `queued`로 바뀌고, 측정 타이머가 적응형(약 6초) 측정을 시작한다.

## 측정 알고리즘

일반 URL은 HTTP `Date` 헤더를 사용한다.

- `HEAD` 요청 우선
- 실패 시 `GET` + `Range: bytes=0-0`
- 각 샘플에서 요청 RTT, 수신 시각, Date 헤더 시각을 기록
- Date 헤더가 다음 초로 넘어가는 경계를 찾아 offset 후보 계산
- 후보 중앙값을 최종 offset으로 사용
- 경계를 찾지 못하면 upper-envelope 방식으로 폴백

네이버 URL은 별도 API를 사용한다.

- `naver.com` 호스트면 네이버 시계 API 호출
- 밀리초 포함 KST 시각을 UTC epoch ms로 변환
- RTT/2 보정
- 저지터 샘플의 중앙값 사용

## 상태 전환

초기 기본 실행은 `idle`이다. URL이 입력되면 `queued`가 되고, 타이머가 시작되면 `measuring`이 된다. 측정 성공 시 `ok`, 실패 시 `failed`가 된다.

기존 대상 재측정은 기존 offset과 새 offset의 차이를 검사한다. 차이가 100ms 이내면 새 값을 반영하고, 초과하면 한 번 더 측정한다. 두 번 모두 100ms를 초과하면 기존 값을 유지한다.

대상 URL이 바뀐 경우에는 기존 offset과 비교하지 않는다. 새 서버의 offset이 크게 다른 것은 정상이다.

## 프론트엔드 동작

UI는 네이버 시계 스타일의 원형 진행 표시, 시각, 날짜, 측정 통계를 보여준다. 사용자는 상단 입력창에서 대상 URL을 바꿀 수 있다.

`clock.js`는 `/api/state`를 1초마다 가져온다. 서버 응답에 담긴 `pcSendTimeAtMs + offsetMs`를 기준으로 잡고, 이후 화면 갱신은 브라우저 `performance.now()`로 계산한다. 작은 offset 변경은 즉시 점프시키지 않고 천천히 보정한다.

## 테스트

테스트는 Pester 기반이다.

```
Invoke-Pester -Script .\tests -PassThru
```

현재 테스트 범위:

- monotonic anchor 초기화와 증가성
- RFC 1123 `Date` 헤더 파싱
- Cristian offset 계산
- 중앙값, 표준편차, 샘플 축소 알고리즘
- NTP timestamp 파싱
- URL normalize helper 일부

## 현재 한계

- 외부 배포형 서비스가 아니며, 로컬 Windows 실행을 전제로 한다.
- `ServerTimeProbe.exe`는 단독 실행 파일이 아니라 런처다. 같은 위치에 `src` 폴더가 있어야 한다.
- URL 입력은 로컬 도구 기준으로 검증한다. 공용 서비스로 바꾸려면 SSRF 방어가 필수다.
- 대상 서버가 정확한 `Date` 헤더를 주지 않으면 정확도도 떨어진다.
- keep-alive 재사용 최적화는 아직 없다.
- 대규모 실측 분석 자동화는 아직 없다.
- `CLAUDE.md`에는 과거 NAS/Node.js 장기 설계 내용이 함께 남아 있어, 현재 구현 설명은 이 문서를 우선한다.
