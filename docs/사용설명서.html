서버시간 측정 도구 사용설명서

웹사이트의 HTTP 서버시간을 재서 로컬 브라우저에 실시간 시계로 보여주는 Windows용 도구입니다.

목차

  1. 실행하기
  2. 측정하기
  3. 화면 보는 법
  4. 재측정 버튼
  5. 빠른 측정 버튼
  6. 인터파크 입력
  7. 문제 해결
  8. 주의사항

1. 실행하기

  1. ServerTimeProbe.exe를 실행합니다.
  2. 브라우저가 자동으로 http://127.0.0.1:8765/를 엽니다.
  3. 검은 PowerShell 창은 닫지 않습니다. 닫으면 측정 서버도 종료됩니다.
종료할 때는 PowerShell 창에서 Ctrl+C를 누르거나 창을 닫으면 됩니다.

2. 측정하기

  1. 상단 입력창에 사이트 주소를 넣습니다.
  2. 측정 버튼을 누릅니다.
  3. 기본 측정 구간은 약 6초입니다. 다만 사전 확인, 느린 응답, 재시도, 추가 샘플링 때문에 실제 완료까지는 더 걸릴 수 있습니다.
  4. 완료되면 큰 시계가 해당 사이트의 추정 서버시간으로 움직입니다.
측정 종류별 최대 시간(버튼 옆 안내줄에도 표시): 측정 최대 20초 · 재측정 최대 15초 · 빠른 측정 최대 5초. 더 오래 걸리는 쪽일수록 더 정확합니다.

주소 입력 예시

ticket.interpark.com
nol.interpark.com
http://den08.inames.kr/
https://naver.com/

주소 앞의 https://는 생략해도 됩니다. 다만 HTTPS 인증서가 맞지 않는 오래된 사이트는 http://로 직접 입력해야 할 수 있습니다.

3. 화면 보는 법

표시뜻
큰 시계현재 추정 서버시간입니다.
측정: N초 전마지막 측정 완료 후 지난 시간입니다.
RTT내 PC와 대상 사이트 사이의 연결 지연입니다. 작고 안정적일수록 좋습니다.
일치폭 ±Nms측정값들이 서로 얼마나 비슷했는지입니다. 작을수록 안정적인 측정입니다.
샘플 N개측정에 사용한 요청 개수입니다.
안전마진너무 빠르게 표시하지 않기 위해 일부러 늦춰 표시하는 값입니다.
중요: 일치폭 ±Nms가 작아도 항상 절대정확하다는 뜻은 아닙니다. 네트워크 지연이 한쪽으로 치우치면 모든 샘플이 같이 밀릴 수 있습니다.

4. 재측정 버튼

재측정은 현재 사이트를 다시 측정합니다. F5와 비슷하지만 페이지 전체를 새로고침하지 않습니다.

기대 동작

  1. 버튼을 누르면 화면이 재측정 중...으로 바뀝니다.
  2. 기존 시계 기준은 임시로 비워집니다. 이전에 틀린 값이 계속 보이지 않게 하기 위해서입니다.
  3. 새 측정이 성공하면 기존값과 비교한 뒤 반영 여부를 정합니다.
  4. 새 측정이 실패하면 기존 측정값은 유지하고 실패 사유만 남깁니다.

재측정도 새로 샘플을 모으는 과정입니다. 기본 측정 구간은 약 6초이고, 최대 15초 안에서 끊습니다. 티켓 오픈 직전에는 누르지 않는 것이 좋습니다.

반영 기준

  • 측정에 쓸 만한 신호(edge)가 5개 미만이면 실패로 보고 기존값을 유지합니다.
  • 기존값과 차이가 30ms 이하이면 흔들림으로 보고 기존값을 유지합니다.
  • 31ms~100ms 차이면 새 측정값을 반영합니다.
  • 100ms 초과 차이면 한 번 더 측정합니다. 두 번째도 100ms를 넘으면 기존값을 유지합니다.
  • 처음 측정하거나 새 URL로 바뀐 경우에는 비교할 기존값이 없으므로 새 측정값을 반영합니다.

5. 빠른 측정 버튼

⚡ 빠른 측정은 정확도를 많이 포기하는 대신 최대 5초 안에 끝내는 측정입니다. 20초를 기다리기 어려울 때 씁니다.

일반 측정과의 차이

측정 / 재측정빠른 측정
최대 시간20초 / 15초5초
정확도(대략)±30~80ms±100~300ms(신호 없으면 ±500ms)
실패 시 처리edge 부족이면 기존값 유지결과가 적어도 그대로 반영(덮어쓰기)

기대 동작

  1. 버튼을 누르면 화면이 빠른 측정 중... (최대 5초)으로 바뀝니다.
  2. 최근에 측정한 값(5분 이내)이 있으면 "덮어쓸까요?" 확인창이 먼저 뜹니다. 확인을 눌러야 진행됩니다.
  3. 측정이 오래되었거나 아직 측정 전이면 확인 없이 바로 측정합니다.
  4. 완료되면 통계줄에 ⚡ 빠른 측정 · … (정확도 낮음)으로 표시됩니다.
빠른 측정은 정밀 측정값이 있어도 더 부정확한 값으로 덮어씁니다. 안전장치는 확인창과 "정확도 낮음" 표시뿐입니다. 티켓 오픈 직전에는 쓰지 마세요. 정확도가 필요하면 일반 측정을 쓰세요.

빠른 측정 다음에 F5로 새로고침하면 다시 일반(15초) 정밀 측정 방식으로 돌아갑니다.

6. 인터파크 입력

인터파크 티켓 관련 주소는 자동으로 같은 측정 주소로 정리됩니다.

입력 가능:
ticket.interpark.com
tickets.interpark.com
nol.interpark.com

실제 측정 주소:
https://nol.interpark.com/ticket

매번 새로 측정하기 위해 뒤에 임의 값이 붙습니다.

https://nol.interpark.com/ticket?t=<random>

화면 아래 통계줄에 측정경로: nol.interpark.com/ticket이 보이면 이 규칙이 적용된 상태입니다.

더 자세한 조사 기록은 docs/인터파크_측정_정리.html에 정리되어 있습니다.

7. 문제 해결

측정 실패가 뜹니다

  • 주소가 잘못됐는지 확인합니다.
  • HTTPS가 실패하는 오래된 사이트라면 http://로 다시 입력합니다.
  • 대상 사이트가 서버시간 정보를 주지 않으면 측정할 수 없습니다.

재측정 버튼을 눌렀는데 바로 시간이 안 바뀝니다

  • 정상입니다. 새 샘플링이 끝나야 새 값이 반영됩니다.
  • 재측정 중에는 기존에 틀린 시계가 계속 보이지 않도록 기준 시계를 비웁니다.
  • 계속 이상하면 브라우저에서 Ctrl+F5를 한 번 누릅니다.

측정 시간이 오래 걸립니다

  • 대상 사이트가 느리게 응답하거나 일부 요청이 timeout되면 측정 시간이 길어질 수 있습니다.
  • 해외 서버, 오래된 사이트, 접속자가 몰린 티켓 사이트는 더 오래 걸릴 수 있습니다.
  • 30초 이상 계속 멈춘 것처럼 보이면 주소를 다시 확인하고 한 번 더 측정하세요.

네이비즘과 1초 안팎 차이가 납니다

  • 우리 도구는 내 PC에서 대상 사이트에 직접 요청해서 시간을 계산합니다.
  • 네이비즘은 네이비즘 서버 쪽에서 측정한 값을 보여주므로 요청 경로와 보정 방식이 다를 수 있습니다.
  • 티켓 사이트는 최종으로 열리는 서비스 주소를 맞춰 측정하는 것이 중요합니다.

PowerShell 창을 여러 개 띄웠습니다

중복 실행하면 어떤 서버를 보고 있는지 헷갈릴 수 있습니다. PowerShell 창을 모두 닫고 ServerTimeProbe.exe를 하나만 다시 실행하세요.

8. 주의사항

  • 이 도구는 대상 사이트가 응답에 담아 보내는 서버시간을 기준으로 합니다. 실제 예매/신청 판정 서버가 같은 시계를 쓴다는 보장은 없습니다.
  • 네트워크 지연이 불안정하면 오차가 커질 수 있습니다.
  • 측정 직후 값이 가장 좋습니다. 오래 켜둔 상태라면 미리 재측정하세요.
  • 티켓 오픈 직전에는 재측정하지 말고 이미 측정된 상태를 유지하세요.
